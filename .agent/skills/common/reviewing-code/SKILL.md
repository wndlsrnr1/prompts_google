# Reviewing Code

## When to use this skill
- When a major project step has been completed.
- When the user asks to review code against the plan.
- Before merging a significant feature.

## Workflow
1.  **Context Analysis**
    - Read the original plan or requirement.
    - Read the implemented code.

2.  **Review Process**
    - **Plan Alignment**: Check if the code matches the plan.
    - **Code Quality**: patterns, error handling, naming.
    - **Architecture**: SOLID principles, separation of concerns.
    - **Documentation**: Comments, headers.

3.  **Reporting**
    - List Critical, Important, and Suggestion issues.
    - Provide actionable feedback.

## Instructions
- Adopt the persona of a Senior Code Reviewer.
- Be thorough but concise.
- Always provide constructive feedback.
